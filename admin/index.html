            Règles du cube

            Consolidation

            L’élément sur lequel vous avez cliqué dans l’arbre est le parent : il devient consolidé si besoin. Cochez les éléments de base à lui rattacher comme enfants (aucun nom à saisir : l’élément parent reste le même ; le serveur exécute la requête d’ajout d’enfants).

            Utilisateurs Palo

            Palo ne définit pas un cube « utilisateur / rôle » sous ce nom : les rôles sont liés aux groupes dans le cube #_GROUP_ROLE (axes groupe × rôle), et les utilisateurs aux groupes dans #_USER_GROUP (utilisateur × groupe). Le cube #_ROLE_RIGHT_OBJECT relie les rôles aux objets de droit. Tout cela se trouve dans la base System (colonne de gauche), pas dans vos bases métier.

            Comptes dans la base System (dimension #_USER_). Les comptes système (admin, _internal_suite, etc.) ne peuvent pas être supprimés ici.
